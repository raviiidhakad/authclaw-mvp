# E2.3 Streaming Hardening Closeout

Date: 2026-06-29

Status: RELEASE CANDIDATE WITH DOCUMENTED PERFORMANCE FOLLOW-UP

Scope: Epic E2.3 final production-hardening and closeout for the AuthClaw streaming pipeline.

## Executive Summary

E2.3 replaces the prior strict buffered streaming internals with an incremental,
security-aware pipeline while preserving the existing Gateway API, provider
contracts, audit contracts, E2.1 reversible tokenization behavior, and E2.2 YAML
plus OPA enforcement semantics.

The active StreamingEngine path now uses:

1. UTF-8 incremental decoding
2. SSE parsing
3. bounded streaming state windows
4. E2.2 policy evaluation
5. E2.1 tokenization/redaction
6. existing OpenAI-compatible SSE output

Focused closeout verification passed with 193 tests passing and 1 live-provider
test intentionally skipped. Docker Compose configuration validated cleanly.

Full backend-suite execution was also attempted during Phase 7. E2.3 tests
passed inside that run, but the full suite was not green because older
integration/sprint suites failed with local infrastructure and Windows
temp-directory setup issues outside the streaming scope.

## Architecture

| Component | File | Responsibility |
| --- | --- | --- |
| Streaming contracts | `apps/api/app/core/engine/streaming_contracts.py` | Stable interfaces and invariants for streaming parser, decoder, policy, tokenization, and emitter boundaries. |
| UTF-8 decoder | `apps/api/app/core/engine/utf8_decoder.py` | Reconstructs complete Unicode text from arbitrary byte chunks without replacement-character leakage. |
| SSE parser | `apps/api/app/core/engine/sse_parser.py` | Parses decoded Server-Sent Events incrementally and fails closed on malformed framing. |
| State machine | `apps/api/app/core/engine/streaming_state_machine.py` | Holds bounded unsafe suffix windows and emits safe text windows in order. |
| Streaming engine integration | `apps/api/app/core/engine/streaming.py` | Connects the pipeline to existing StreamingEngine behavior and preserves API/output compatibility. |

## Runtime Pipeline

Provider byte chunks flow through:

```text
provider bytes
  -> Utf8IncrementalDecoder
  -> SseParser
  -> StreamingRedactionStateMachine
  -> policy evaluation
  -> tokenization/redaction
  -> existing OpenAI-compatible SSE output
```

The implementation remains fail-closed: malformed UTF-8, malformed SSE,
interrupted provider streams, policy-deny decisions, and scanner failures do not
release unsafe partial content.
