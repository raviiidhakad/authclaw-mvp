# E2.3 Streaming Hardening Phase 1

Phase 1 is a non-runtime preparation phase. It formalizes the streaming
contracts, preserves the v0.8 architecture, and defines the tests required
before token-by-token streaming behavior is implemented.

## Project Plan Requirements

- Gateway traffic must redact sensitive data in real time, including streaming responses.
- Gateway scope includes streaming-safe filtering.
- NFR-1.2 requires token-by-token streaming filtering with no fragmentation.
- E2.3 requires token-by-token chunk evaluation and back-pressure handling.
- Phase 2 exit criteria require streamed sensitive data to be redacted and policies to block disallowed traffic before egress.

## Current Architecture Audit

| Area | Current state | Must remain unchanged in Phase 1 |
| --- | --- | --- |
| Gateway streaming flow | Gateway rejects passthrough streaming and routes supported streaming requests through `StreamingEngine`. | No gateway execution changes. |
| StreamingEngine | Collects provider stream deltas, scans the assembled response, emits sanitized SSE, and publishes stream audit events. | No behavior changes. |
| Provider adapters | Expose `stream_response(response)` and normalize provider chunks into OpenAI-compatible SSE bytes. | No adapter changes. |
| StreamingBuffer | Existing sliding-window helper detects PII across text chunk boundaries and fails closed on scanner errors. | No behavior changes. |
| Tokenization | Gateway uses E2.1 token vault helpers for reversible placeholders and outbound detokenization. | Preserve tenant-scoped tokenization. |
| OPA/policy | Gateway uses authoritative YAML + OPA adapter enforcement before provider egress. | Preserve E2.2 enforcement. |
| Audit | Streaming emits started/completed/failed events and raw gateway audit retention is disabled by default. | Preserve sanitized audit behavior. |

## Contracts

The contract module defines responsibilities only:

- `SseParserContract`: parse provider bytes into normalized SSE events.
- `Utf8IncrementalDecoderContract`: decode bytes without corrupting split UTF-8 codepoints.
- `StreamingRedactionStateMachineContract`: hold unsafe suffixes and release only safe text windows.
- `StreamingPolicyEvaluationContract`: evaluate sanitized windows through the existing policy boundary.
- `StreamingTokenizationContract`: apply E2.1-compatible tokenization/redaction without raw value exposure.
- `StreamingOutputEmitterContract`: emit OpenAI-compatible SSE from sanitized payloads.

## Architecture Invariants

- Preserve Gateway API and response contracts.
- Preserve provider adapter abstraction.
- Preserve OpenAI-compatible SSE.
- Preserve E2.1 reversible tokenization.
- Preserve E2.2 YAML + OPA enforcement.
- Preserve sanitized audit behavior.
- Preserve fail-closed security posture.

## Future Test Plan

| Test class | Coverage |
| --- | --- |
| Unit | SSE parsing, UTF-8 decoding, rolling window, redaction state machine, output emitter. |
| Integration | Gateway -> provider stream -> parser -> redaction -> policy -> emitter -> audit. |
| Gateway | Passthrough remains blocked, unsupported modes fail safely, route policy blocks before egress. |
| Provider contract | OpenAI/Groq, Anthropic, Cohere, Azure OpenAI stream normalization. |
| Boundary | PII split across chunks, SSE events, JSON fragments, UTF-8 codepoints, and final flush. |
| Policy | Allow, redact, block, STRICT failure, COMPATIBILITY fallback where applicable. |
| Tokenization | Mask/hash/synthetic/reversible paths never expose raw sensitive values in audit/UI/logs. |
| Performance | First-token latency, per-chunk overhead, p50/p95/p99, memory per stream. |
| Cancellation | Client disconnect, upstream timeout, upstream disconnect, partial audit finalization. |
| Back-pressure | Bounded buffers, slow client, fast provider, no token loss or reordering. |

## Risks And Mitigations

| Risk | Mitigation |
| --- | --- |
| Sensitive text split across chunks leaks before detection. | Hold retained suffix and scan safe prefix before emission. |
| UTF-8 split corrupts content or hides PII. | Use incremental decoder and fail closed on invalid sequences. |
| SSE parser breaks provider compatibility. | Provider contract fixtures for each supported adapter. |
| Streaming redaction adds too much latency. | Benchmark per-chunk overhead and tune bounded window size. |
| Policy behavior diverges between streaming and non-streaming. | Reuse existing E2.2 adapter boundary and shared decision metadata. |
| Reversible tokenization leaks raw values during detokenization. | Make outbound detokenization explicit, tenant-scoped, and policy-gated. |
| Client disconnect leaves incomplete audit. | Emit sanitized failed/cancelled stream event with partial hashes only. |
| Back-pressure drops or reorders chunks. | Bounded queues, ordered sequence numbers, cancellation tests. |

## Acceptance Criteria

- Phase 1: contracts exist, invariants are explicit, tests prove no runtime coupling.
- Phase 2: parser handles SSE comments, multiline data, `[DONE]`, malformed events, and split UTF-8.
- Phase 3: rolling window redacts entities split across provider chunks without releasing raw text.
- Phase 4: streaming enforcement uses existing tokenization and policy boundaries without schema/API changes.
- Phase 5: gateway returns OpenAI-compatible token-by-token SSE with passthrough still disabled.
- Phase 6: provider-specific streaming tests pass and all failures emit sanitized fail-closed responses.
- Phase 7: benchmark records first-token and per-chunk overhead against the Project Plan latency target.
