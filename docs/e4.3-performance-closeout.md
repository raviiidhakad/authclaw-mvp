## Executive Summary

E4.3 establishes an evidence-producing benchmark system for AuthClaw v0.10.0.
The implementation covers:

1. Immutable performance benchmark contracts.
2. A reusable benchmark harness and measurement framework.
3. Gateway benchmark scenarios.
4. Streaming benchmark scenarios.
5. Audit Export benchmark scenarios.
6. Load, concurrency, and resource profiling scenarios.
7. Consolidated performance closeout reporting.

The local closeout run passed all measured benchmark groups. The results are
valid for local mocked/in-memory benchmarking only. Production capacity,
provider network behavior, production database latency, Redis, ClickHouse,
Vault, and production signing/HSM latency remain deployment-stage validation
items.

Verdict: E4.3 COMPLETE.

## Known Benchmark Limitations

| Limitation | Impact | Follow-up |
| --- | --- | --- |
| Mocked providers | Does not include provider network/API latency. | Run provider-backed staging benchmarks. |
| In-memory audit repository | Does not include PostgreSQL/ClickHouse query and write-path costs. | Run large exports against staging data stores. |
| Deterministic local signer | Does not include KMS/HSM or certificate operations. | Benchmark production signer once available. |
| c5 closeout load smoke | Does not prove c25/c50/c100 production capacity. | Run full load profile in staging. |
| Local Windows process measurements | CPU/memory values reflect this development machine only. | Capture Linux container/staging measurements before production SLOs. |
| No optimization in E4.3 | E4.3 is measurement-only by design. | Use benchmark evidence to prioritize a future optimization track. |
